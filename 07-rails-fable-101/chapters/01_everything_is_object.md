# 第1章 原石に触れる — すべてがオブジェクト

## 💎 今日のお話

山あいの町の宝石工房「紅玉堂」。あなたは今日から見習い職人です。
親方が最初に手渡してくれたのは、彫刻刀ではなく一粒の原石でした。

「まず知ってほしいのは、この工房では **すべてが宝石(オブジェクト)** だということだ。
数も、文字も、『何もない』ことすらもな」

Python でも「すべてがオブジェクト」と聞きました。Go では struct と関数の世界でした。
Ruby はその点で **Python よりさらに徹底しています**。まずは対話環境 `irb` で確かめましょう。

```bash
irb
```

## すべてがオブジェクト — 本当に、すべてが

```ruby
1.class          # => Integer
"ruby".class     # => String
nil.class        # => NilClass
true.class       # => TrueClass
Integer.class    # => Class(クラスすらオブジェクト)

# だから「値そのもの」にメソッドが生えている
-5.abs           # => 5
10.even?         # => true
"kogyoku".length # => 7
"ruby".upcase    # => "RUBY"
3.times { puts "研磨!" }  # 研磨! が 3 回表示される
```

最後の `3.times` を見てください。**整数の 3 に「3 回繰り返す」というメソッドがある**。
これが Ruby の感覚です。制御構造すらオブジェクトへのメッセージとして書けます。

> 🐍 **Python との違い①: len(s) か s.length か**
> Python では長さは `len(s)` と **関数** で取りました(`__len__` に委譲される設計)。
> Ruby では `s.length` と **オブジェクト自身に聞きます**。Ruby には
> 「トップレベルの組み込み関数でデータを操作する」文化がほぼなく、
> 操作は原則すべて **レシーバのメソッド** です。この一貫性が、後述の
> メソッドチェーンの読みやすさ(左から右に処理が流れる)につながります。

> 🐹 **Go との違い①: 宣言なし・ゼロ値なし**
> Go では `var x int` と宣言し、未初期化でもゼロ値が入りました。Ruby は Python と同じく
> **代入した瞬間に変数が生まれ**、未定義の変数に触ると `NameError` で即座に落ちます。
> 「黙ってゼロで動き続ける」Go 的な事故は起きない代わりに、typo は実行するまで分かりません。
> 静的検査を型に頼れない分、Ruby はテスト文化が発達しました(第14章の伏線です)。

## 変数と命名

```ruby
stone = "garnet"        # ローカル変数: snake_case(Python と同じ)
price = 4800
PI_RATIO = 3.14159      # 大文字始まりは定数(再代入すると警告が出る)

stone = 42              # 動的型付け: 型はいつでも変わる(推奨はしない)
```

Ruby の変数は **オブジェクトへの参照(ラベル)** です。Python とまったく同じメンタルモデル、
Go でいえば「全部ポインタ」に近い世界です。

```ruby
a = "ruby"
b = a
b << " on rails"   # << は破壊的な追記
puts a             # => "ruby on rails"(a も変わる!同じ実体を指しているから)
```

## 文字列 — 式展開と 2 種類のクォート

```ruby
name = "紅玉堂"
puts "ようこそ #{name} へ"      # ダブルクォート: #{} で式展開
puts 'ようこそ #{name} へ'      # シングルクォート: 展開されずそのまま表示

puts "在庫: #{3 * 4} 個"        # 中は任意の式でよい
```

Python の f-string は先頭に `f` が必要でしたが、Ruby のダブルクォートは **常に展開可能** です。
迷ったらダブルクォートで書けば困りません。

```ruby
"garnet".sub("net", "den")   # => "garden"(最初の 1 箇所を置換)
"a,b,c".split(",")           # => ["a", "b", "c"]
"  ruby  ".strip             # => "ruby"
"ruby" * 3                   # => "rubyrubyruby"(Python と同じ!)
```

## シンボル — Rails 理解の最重要アイテム

Ruby には文字列とよく似た **シンボル** という型があります。先頭にコロンを付けます。

```ruby
:ruby.class        # => Symbol
:name == :name     # => true
"name" == "name"   # => true(結果は同じだが…)

:name.object_id == :name.object_id   # => true(同じシンボルは常に同一オブジェクト)
"name".object_id == "name".object_id # => false(文字列は書くたびに新しく作られる)
```

使い分けの感覚はこうです:

- **文字列** — 内容そのものが大事なデータ(商品名、顧客のコメント)
- **シンボル** — プログラム内部の「名札」「識別子」(メソッド名、ハッシュのキー、設定項目名)

なぜこれが最重要かというと、**Rails のコードはシンボルだらけ** だからです。

```ruby
# 第10章で書くことになる Rails のコード(今は読めなくてOK)
validates :name, presence: true
has_many :orders
```

`:name` も `:orders` も「name という属性」「orders という関連」を指す **名札** です。
これが文字列でなくシンボルなのは、同じ名札を何万回使ってもオブジェクトが 1 つで済み、
比較も一瞬(同一性比較)だからです。

> 🔍 **なぜそうなっているの? — Ruby の生い立ちと「快適さ」**
> Ruby は 1995 年、日本人の **まつもとゆきひろ(Matz)** 氏が公開した言語です。
> 設計の核は Smalltalk の **純粋オブジェクト指向**(すべてがオブジェクトで、
> 計算は「オブジェクトへのメッセージ送信」)と、Perl の実用性・書き味の融合でした。
> Matz 氏の設計原理は有名な一言に集約されます —
> **「Ruby は書いていて楽しい(プログラマの幸福のための)言語」**。
> 機械の効率(Go)やただ一つの正しい書き方(Python の The Zen)より、
> **書き手の気持ちよさと表現の自由** を優先する。この価値観が、この教材で繰り返し
> 出てくる Ruby のあらゆる設計判断の根っこにあります。

## nil — 「何もない」もオブジェクト

```ruby
box = nil
box.class      # => NilClass
box.nil?       # => true
box.to_s       # => ""(空文字に変換できる)
box.upcase     # => NoMethodError! nil に upcase はない
```

Go の `nil` はメソッドを呼べば panic、Python の `None` は `AttributeError` でした。
Ruby の `nil` も `NoMethodError` を出しますが、**nil 自身がオブジェクトなので
`nil?` や `to_s` のようなメソッドは普通に呼べます**。さらに Ruby には
安全呼び出し演算子があります:

```ruby
box&.upcase    # => nil(box が nil なら、呼ばずに nil を返す)
```

TypeScript の `?.` と同じ発想です(Ruby が 2015 年に先行導入し「ぼっち演算子」——
`&.` が体育座りに見える——という愛称で呼ばれています)。

## 数値 — Python 似の安心設計

```ruby
10 / 3        # => 3(整数同士は整数除算。Python 2 と同じで Python 3 と違う!注意)
10.0 / 3      # => 3.3333...
10 % 3        # => 1
2**100        # => 1267650600228229401496703205376(多倍長整数!Python と同じ)
1_000_000     # 桁区切りの _ が使える(Go と同じ)
```

Go では `int` がオーバーフローしましたが、Ruby の整数は Python 同様
**自動的に多倍長** になります。ただし除算は Python 3 と挙動が違う点に注意してください
(`10 / 3` は `3` です)。

## 出力の道具: puts / p / pp

```ruby
puts "紅玉"     # 人間向け表示(to_s)。Python の print に相当
p    "紅玉"     # デバッグ向け表示(inspect)→ "紅玉" とクォート付きで出る
pp   {a: 1}     # 複雑な構造を整形表示(Python の pprint に相当)
```

`p` は **引数をそのまま返す** ので、式の途中に差し込んでデバッグできます。
「この値、今なんだ?」と思ったら `p` を挟む——工房の虫眼鏡です。
